## Why
The portfolio now has its permanent domain and basic analytics, but it still lacks the technical SEO foundation needed to win branded search cleanly and give search engines a strong understanding of Dushyant, his work, and the canonical site.

Right now the opportunity is not generic keyword-chasing. It is to make `dushyantgarg.com` the clear official result for Dushyant Garg, his projects, and his writing, while keeping the site fast and human.

## What Changes
- add technical discovery artifacts for crawlers, including sitemap and robots support
- strengthen canonical metadata so every public page resolves to the production domain with page-specific titles and descriptions
- add structured data for portfolio identity, especially `Person` and `WebSite`
- support search engine ownership verification without needing one-off code edits each time
- document the post-merge operator steps for Google Search Console and Bing Webmaster Tools

## Scope
In scope:
- technical SEO and search discoverability foundations inside the Astro portfolio
- metadata, canonicalization, structured data, crawl/index artifacts, and verification hooks
- lightweight documentation for submission and validation

Out of scope:
- broad content strategy
- backlink campaigns
- paid acquisition
- trying to rank for broad generic AI keywords
- adding custom analytics/event tooling outside current plan limits

## Success Criteria
- the site exposes crawl/index artifacts that search engines expect
- the site clearly identifies Dushyant Garg and the portfolio as the canonical web presence
- each public page has purposeful metadata instead of generic defaults
- another agent can implement the work directly from this change without guessing the intended boundary
